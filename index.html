Migrate to Hugo from Jekyll - 2014-03-10 00:00:00

Move static content to static Jekyll has a rule that any directory not starting with _ will be copied as-is to the _site output. Hugo keeps all static content under static. You should therefore move it all there. With Jekyll, something that looked like ▾ <root>/ ▾ images/ logo.png should become ▾ <root>/ ▾ static/ ▾ images/ logo.png Additionally, you’ll want any files that should reside at the root (such as CNAME) to be moved to static.

写在前面 - 2017-11-05 17:40:55

去年毕业，到现在已经一年有余了。 毕业的时候不想去电网，觉着电网混吃等死，没有意义。像我这样的穷小子，本来就什么都没有，索性就任性的闯一闯，反正结果也不会差到哪里去-自己养活自己肯定还是没问题的。 于是来到了台达。 前两个月过的还算顺畅，基本没有安排任务，就是把电路图拿出来看了一遍。说是看了一遍，起始很多地方都没有细算，大体的缕一缕结构就放到那了。 后来忙了一点点，但关键性的活都不是我做，我就负责给机器上电，看一下工作或者故障情况，如果故障了，把机器拆开，放平，关键信号用探头勾出来，让老工程师分析，跟着学习。 然后是解Thermal，把变压器，电感，半导体的温度用热耦线点出来，放在Chamber里跑，看最高能到多少度，能不能满足derating。变压器core热或者wire热，我都不知道如何处理，只能拿着数据去找老大，让他给建议。按他的建议，再点热耦线，再测，反反复复测了很久。 进步的很缓慢，我甚至觉着自己什么都没干，公司就每个月给我发了工资。工作的积极性呢，说高不高，总觉着自己没什么实力，做什么事情都不敢自己拿决定，要去咨询老工程师，得到他们的首肯，才敢做。老工程师们就开始变的不耐烦：同事帮你是情谊，不帮你也正常。 天天做这些事儿，总觉着不踏实，人是飘着的，说是电源工程师，既不熟悉拓扑，也不熟悉器件，管子不会选，变压器电感不会设计，控制电路也是一踏糊涂。 而且我发现，很多人对这些都是一知半解。。 但这并不妨碍他们做电源，一个部门里有上两三个头脑清醒，思维清晰的人，就可以撑着往前运转了。 如果决定都让别人做，你的作用是什么呢，不动脑子，机械的执行？执行力固然重要，但背后的思维更重要，这也是为什么会有这个网站的原因。
