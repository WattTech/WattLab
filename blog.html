在Windows下利用coding.net搭建基于Pelican的个人博客

搭建个人博客

搭建一个网站一般需要三个部分：1，域名和解析。2，主机。3，网站框架。 域名现在可以在万网http://net.cn 购买（被阿里收购了，现在名叫阿里云），同时阿里云也提供主机，域名解析服务。网站框架个人博客使用最著名的就是wordpress。 github的出现，催生了一种新的个人博客搭建方法，利用github pages提供jekyII搭建静态博客，省掉了购买主机的费用。而且github提供绑定域名的服务，可以实现很好的访问体验。 由于国内特殊的网络情况，github时不时就有被墙的风险，国内模仿github的网站主要有gitcafe，而gitcafe被coding.net收购了。coding.net是国内的服务提供商，只要公司不倒掉，访问速度和访问可达性还是比较有保障的。 因此，我决定使用coding.net搭建一个个人blog。

网上给出的静态博客系统一般有

  1. 1. Simple 使用简单，登录 http://isnowfy.github.io/simple/ ，用github帐号登录。但这个是在github上托管的，速度有影响，而且可能会被墙掉。
  2. 2. Logecho号称是一天撸出来的...
  3. 3. Hexo基于node.js，不懂
  4. 4. JekyII和Octopress基于Ruby，还是不懂
  5. 5. Pelican基于Python，现在已经迭代到3.6了，有人维护debug，看起来还不错。 于是我选择了Pelican，网上很多给出的教程都是基于Linux的，而一般人都使用Windows，虽然以前清华的大神发文说可以完全使用Linux工作，但对于普通人，这完全是不必要的。

在conding.net的设置

参考官方给的方法 https://coding.net/help/doc/pages/index.html，设置好之后，可以看到“hello coding”的网页，说明就成功了。此处git使用的文件路径，不用和Pelican的相同。

独立域名

如果使用独立域名，需要在购买域名的提供商处设置解析。 解析设置：记录类型:CNAME；主机记录：WWW；解析线路：默认；记录值：xxx.coding.me; MX优先级、TTL默认既可。记录值务必填写coding给出的数值。

在Windows下使用pelican在coding.net搭建blog。

此处的做法是不唯一的，很多方式都是互通，大家仅作参考。

  1. 1. 安装python，在官网http://python.org 下载安装包，然后双击，注意点上最下面的“添加环境变量”，后面的一路确定就行了。 'pip is already installed if you're using Python 2 >=2.7.9 or Python 3 >=3.4 downloaded from python.org, but you'll need to upgrade pip.
  2. 2. 安装Git，在Git官网http://git-scm.com 下载安装包，默认安装既可。
  3. 3. 升级pip，安装新版本的Python时，默认是安装了pip的，但是需要更新。 在Windows系统下，双击Git图标，然后输入 'python -m pip install -U pip' 会出现进度条，安装完成后会有提示。
  4. 4. 安装pelican和markdown。 仍然在刚才打开的对话框输入 'pip install pelican markdown' 会出现进度条，分别安装这两个软件
  5. 5. 至此，Pelican在Windows上就安装好了，现在找个地方新建一个文件件，用来在本地存放你的网站，此处以f盘mysite为例。这个过程是可以直接在我的电脑里操作的。 还是在刚才的对话框里输入，使用命令进入建好的文件夹 'cd f:/mysite'
  6. 6. 配置Pelican，在刚才的命令行里输入 'pelican-quickstart' 除了SITENAME配置项外，其他均可选择默认，在后绪的开发过程中，可以在pelicanconf.py中进行修改
  7. 7. 执行完上一步之后，mysite文件夹里会出现一些新文件。这里支出两个，一个是content文件夹，所有用markdown写的东西(.md)，必须放在这个文件夹里。output文件夹是使用Pelican生成的，用于上传到主机的文件。

主要注意一下几点。

  1. 1. 在output文件夹里初始化Git仓库，添加coding.net完远程仓库之后，先pull一下，将coding上的文件取回
  2. 2. 再在content里面用markdown写文章，写好之后保存。
  3. 3. 在Git命令行中输入pelican content，注意此处的路径是在mysite文件夹下，而不是output文件夹下。
  4. 4. 生成完成后，进入output文件夹，然后使用Git推送到远程仓库。

Git

1. 使用Git使用参考：http://www.liaoxuefeng.com/ 这个网站给出了详细的git教程。

2. 然后打开浏览器，查看自己的网站吧。
